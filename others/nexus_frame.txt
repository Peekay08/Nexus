i. Qt (GUI Framework)
Purpose: Build cross-platform user interfaces.
How it works: Qt provides widgets, layouts, and system access (like file dialogs, window control, drag-and-drop, etc.). It compiles natively for Windows, macOS, and Linux.
Why it's good for NEXUS:
Makes the UI responsive and consistent across OSes.
Lets us build things like the App Launcher, File Explorer, Task Manager UI, Settings Panel, etc.
Built-in support for high-DPI, touch, themes, and localization.

ii. Boost (C++ Utility Library)
Purpose: Add powerful features to C++ without rewriting common code.
How it works: Boost is a collection of high-performance libraries for:
Filesystem access
Threading/multithreading
Regex, string handling
Graphs, serialization, etc.
Why it's good for NEXUS:
Weâ€™ll use it for managing the file system (copying, scanning apps).
Helps manage multi-user processes and communication between modules.

iii. LLVM/Clang (Compiler Infrastructure)
Purpose: Analyze, compile, and transform source code or binaries.
How it works: LLVM is a compiler framework. Clang is a frontend for C/C++.
Why it's good for NEXUS:
We can use it to decompile or recompile apps across platforms (e.g., convert a Windows C++ app to work on Linux).
Can generate bytecode or machine code specific to an OS.
Helps power the App Rebuilder Module.

iv. CMake (Build System)
Purpose: Make it easier to compile and manage large C++ projects across platforms.
How it works: Uses configuration files (CMakeLists.txt) to automate building, linking, and dependencies.
Why it's good for NEXUS:
Required when using multiple libraries like Qt, Boost, LLVM.
Ensures the app compiles properly on all OSes.

v. libcurl (Networking)
Purpose: Internet communication.
How it works: Lets C++ apps make HTTP/HTTPS requests, upload/download files, use APIs.
Why it's good for NEXUS:
Needed for features like checking for updates, downloading data, and syncing with cloud services or online resources.

vi. (Optional) OpenCV
Purpose: Image and video analysis.
How it works: Processes visual data (like screenshots, UI layouts).
Why it's good for NEXUS (optional):
Can help identify visual components in apps we analyze (e.g., find buttons, layouts, icons).
Useful if you plan to recreate app UIs without source code.
